# Frontend Decisions

更新时间：2026-07-06

本文记录当前用户对 inline TUI / future frontend polish 的取舍偏好。以下判断是 2026-07-06 的当前意见，后续可以随产品体验变化而调整。

## 当前偏好

- 本轮多工具结果列表 / Ctrl+O 选择展开：用户感兴趣，但先不着急推进。之前做过并失败过，说明实现难度和交互细节风险都不低。
- 工具 trace 文案：不要过度拟人化。工具行应简洁、信息化，例如展示搜索词、路径、命令、URL、进程标签；拟人化语气主要留给助手回复正文。
- 确认框视觉层级：可以继续优化。重点是风险、默认动作、允许/拒绝动作和关键预览信息更清楚。
- 长输出折叠：可以做。完整输出展开时作为新的展开块打印在当前 scrollback 位置；不要尝试回到原截断处插入，因为普通终端 scrollback 不适合做历史位置重写。
- 输入区细节：暂时有待讨论，不作为当前优先项。slash command 相关能力后端线也在处理，前端先不重复规划。
- 状态行中文化：没必要，不作为当前目标。

## 当前可推进项

- 继续调整确认框视觉密度和层级。
- 设计长工具输出的前端截断与展开策略。
- 保持工具 trace 简洁化，避免 raw JSON 和过度拟人化描述。
