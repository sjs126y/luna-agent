# Repository Summary

Require an explicit `owner/repo`. Use GitHub MCP tools rather than general web search.

1. Read repository metadata, default branch, README, and top-level tree.
2. Sample recent commits, open issues, and open pull requests; do not exhaustively fetch large lists.
3. Describe purpose, primary technologies, layout, activity, setup path, and current areas of attention.
4. Separate repository facts from inference. Include GitHub URLs or stable identifiers for important claims.
5. Stay read-only. Do not create issues, comments, branches, or pull requests.
