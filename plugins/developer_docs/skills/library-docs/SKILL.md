# Library Documentation

1. Identify the library and requested version. Ask only when ambiguity changes the answer.
2. Resolve the Context7 library identifier before querying documentation.
3. Query the smallest relevant topic; refine once if the result is incomplete.
4. Base API names, options, defaults, and examples on retrieved documentation, not memory.
5. State the version or documentation scope and distinguish documented behavior from inference.
6. Give a minimal working example when useful. Do not fabricate unavailable APIs.
